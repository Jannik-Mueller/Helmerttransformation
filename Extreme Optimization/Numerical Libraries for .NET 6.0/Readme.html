1. Introduction

The Extreme Optimization Numerical Libraries for .NET are a solid foundation for your numerical computing needs on the Microsoft .NET platform.

Its broad base of algorithms covers a wide range of numerical techniques, including: linear algebra (BLAS and LAPACK routines), numerical integration and differentiation, solving equations, complex numbers, and more.

The classes in the Extreme Optimization Numerical Libraries for .NET and the relationships between them match our every-day concepts, and breaks new ground in usability for numerical software development.

We implemented the best algorithms available today to provide you with great performance on a robust, fast toolset.

3. System Requirements

The Extreme Optimization Numerical Libraries for .NET supports Microsoft .NET versions 2.0-4.5. There are specific versions for .NET 2.0, .NET 3.5, and .NET 4.0. Each edition of the Extreme Optimization Numerical Libraries for .NET requires the corresponding .NET framework version to be installed on your system. The .NET framework is available for download from the Microsoft web site at

http://msdn.microsoft.com/netframework/howtoget/default.aspx

The sample solutions and project files were created using Microsoft Visual Studio .NET. We recommend using Microsoft Visual Studio.NET to compile and build the samples. However, the Microsoft .NET framework includes command line tools that enable you to build the applications without the Visual Studio .NET IDE.

The reference guide is in Adobe Acrobat format. You can obtain an Acrobat Reader from

http://www.adobe.com/

To install the Extreme Optimization Numerical Libraries for .NET, simply run the installer in your download directory and follow the installation wizard instructions. You can choose for which .NET edition to install, and for which Visual Studio version to provide samples and other support files.

5. Design-time License

Note: This section applies only to the retail version of the libraries. The trial version does not require a design-time license.

The Extreme Optimization Numerical Libraries for .NET verify that you have a valid license at compile time, and embeds a license key in the compiled assembly. Without this embedded license key, the application will encounter a LicenseException when it is run.

In order for the compiler to find the license information, you must supply a licenses.licx file with the correct license text.

If you are building an ASP.NET application, you need to create a license assembly. You do this by right-clicking on the licenses.licx file in the Solution Explorer and choosing 'Build runtime licenses' from the context menu.

If you are building an Azure WCF Webrole, the licenses.licx file should be located in the Properties folder.

To add the required entry to an existing licenses.licx file manually:

  1. In the Project Explorer, make sure 'Show All Files' is active.
  2. Find the licenses.licx file in the project directory and open it.
  3. Depending on the .NET version, add one of the following lines:
    Extreme.License, Extreme.Numerics.Net20, Version=3.6.10055.0, Culture=neutral, PublicKeyToken=9e513770f58567b2 Extreme.License, Extreme.Numerics.Net35, Version=6.0.16073.0, Culture=neutral, PublicKeyToken=9e513770f58567b2 Extreme.License, Extreme.Numerics.Net40, Version=6.0.16073.0, Culture=neutral, PublicKeyToken=9e513770f58567b2
    without line breaks and without any leading or trailing spaces. Only add the line for the .NET version you are using.
  4. Save the file.

To add a license.licx file to your project manually:

  1. From the Project menu, choose 'Add New Item...'
  2. Expand the tree on the left, and go to the 'Utility' folder.
  3. Click on 'Text File', and type 'licenses.licx' in the Name field.
  4. Click 'Open'.
  5. Depending on the .NET version, add one of the following lines:
    Extreme.License, Extreme.Numerics.Net20, Version=3.6.10055.0, Culture=neutral, PublicKeyToken=9e513770f58567b2 Extreme.License, Extreme.Numerics.Net35, Version=6.0.16073.0, Culture=neutral, PublicKeyToken=9e513770f58567b2 Extreme.License, Extreme.Numerics.Net40, Version=6.0.16073.0, Culture=neutral, PublicKeyToken=9e513770f58567b2
    without line breaks and without any leading or trailing spaces. Only add the line for the .NET version you are using.
  6. Save the file.

After completing these steps, you should be able to compile your application and run it on any machine. Since the license key is embedded in the executable, you only need the license on the development machine.

Note that if you are building a class library project, the applications that use this library must also be able to produce a license. This means that these applications must be built on a machine that has a valid license installed.

When you are upgrading from an earlier version, or any time you install an update of the library that changes the version number, the existing .licenses file becomes invalid. You will have to perform the same procedure again.

When you are building an ASP.NET 2.0+ application, you may have to build an assembly containing the license key manually. After following the steps outlined above for creating a licenses.licx file, you need to do the following:

  1. Right click on licenses.licx in the Solution Explorer and select "Build Runtime Licenses" from the context menu.
  2. This will create a file named App_Licenses.dll in the solution's /bin folder.
  3. Make sure this file is deployed with your application.

7. QuickStart Samples

The Extreme Optimization Numerical Libraries for .NET includes a large number of QuickStart Samples in both C# and Visual Basic.

The QuickStart Samples are installed in

<Program Data>\Extreme Optimization\<VS Version>\QuickStart\CSharp
<Program Data>\Extreme Optimization\<VS Version>\QuickStart\VisualBasic
<Program Data>\Extreme Optimization\<VS Version>\QuickStart\FSharp

All QuickStart Samples have been conveniently collected in a single Visual Studio .NET solution file. The installation wizard creates shortcuts to these solution files in the start menu.

The QuickStart samples are available online at http://www.extremeoptimization.com/quickstart/

The QuickStart Samples are 'Sample Code' as specified in section 5 of the License Agreement.

To build the QuickStart Samples using Visual Studio .NET, follow these steps:

  1. Open the QuickStart Samples Solution file for the language from the of your choice Start Menu.
  2. From the Build menu, choose 'Build solution.'
To run a specific QuickStart Sample:
  1. In the Solution Explorer, right-click on the project you would like to run. Choose 'Set as startup project' from the context menu.
  2. From the Debug menu, choose 'Start'
To build QuickStart Samples from the command line, follow these steps:
  1. Make sure that the .NET Framework 'BINN' directory is in your directory search path (PATH).

  2. From within a command window, change the current directory to a specific QuickStart Sample. For example:

    <Program Data>\Extreme Optimization\<VS2015\QuickStart\CSharp\AdvancedIntegration

  3. Call the appropriate compiler, and include a reference to the Extreme.Numerics main assembly. For example:

    csc /r:Extreme.Numerics.Net40.dll AdvancedIntegration.cs

  4. Run the executable. The name of the executable should be the same as the QuickStart Sample.
